# Visión general — NetBox + Odoo + neo_stack

## Por qué

- Fuente única de verdad (CMDB viva) para red/DC/activos.
- Reduce inconsistencias de inventario (IPs duplicadas, VLANs en conflicto).
- Habilita automatización: datos estructurados alimentan pipelines (neo_stack) y ERP (Odoo).

## Para qué

- Planeación y operación de red/infra.
- Integración con Odoo: activos, inventario, órdenes de servicio, costos.
- Integración con neo_stack: aprovisionamiento, detección de drift, compliance.

## Dolencias resueltas

- Hojas de cálculo sin trazabilidad.
- Cambios sin historial/aprobación.
- Falta de visibilidad de dependencias (servicio → app → host → IP → rack).
- Retrabajo en soporte por datos inconsistentes.
